# Code Audit

This directory contains the cross-cutting audit artifacts for the repository.

## Structure

| Path | Purpose |
|------|---------|
| `README.md` | This file — how to use the audit system |
| `index.md` | Consolidated audit report |
| `plan.md` | Improvement plan derived from audit findings |
| `tooling.md` | Tooling audit (linters, formatters, CI) |
| `test_hints.md` | Test strategy and coverage hints |
| `patterns.md` | Structural pattern index, touch points, and semantic map |
| `files/` | Per-file audit reports |
| `amendments/` | Amendment records (one per code change) |

## Running the Skill

### AUDIT mode (read-only)

```bash
python scripts/repo_audit.py audit
```

Scaffolds `_code_audit/` with placeholder files. The actual content (per-file reports, patterns, etc.) is generated by the repo-audit skill during a Claude Code session.

### AMEND mode (change workflow)

```bash
python scripts/repo_audit.py amend \
    --slug short-name \
    --description "What changed and why" \
    --primary path/to/main_target.py \
    --related path/to/related_file.py \
    --files path/to/changed_file.py path/to/other.toml \
    --patterns P1 P3
```

Creates `_code_audit/amendments/YYYY-MM-DD_<slug>.md` with all required YAML fields and TODO sections to fill in.

**Before filling in the amendment, you MUST read:**
1. `CLAUDE.md` (codebase snapshot)
2. `_code_audit/patterns.md` (pattern lookup table)

## Amendment Protocol

Every commit that touches **code-like files** must include a valid amendment record staged in the same commit.

### Enforced extensions

**Source:** `.py`, `.js`, `.ts`, `.jsx`, `.tsx`, `.sh`, `.bash`, `.zsh`, `.ps1`
**Config:** `.toml`, `.yaml`, `.yml`, `.json`

### Excluded from enforcement

- Files under `_code_audit/` (audit artifacts)
- Files under `.github/` (CI workflows)
- `CLAUDE.md`, `.gitignore`, `LICENSE`
- Markdown files (`.md`) outside `_code_audit/`
- Pure deletions (only additions, copies, modifications, renames are enforced)

### Required YAML fields in amendment record

| Field | Required Value |
|-------|---------------|
| `mode` | `amend` |
| `slug` | Any non-empty string |
| `date` | Any non-empty string (ISO format) |
| `description` | Any non-empty string |
| `snapshot_used` | `CLAUDE.md` |
| `patterns_used` | `_code_audit/patterns.md` |
| `integrity_check_done` | `true` |
| `primary_files` | Non-empty list of primary target files |
| `related_files_considered` | List of related files checked (may be empty) |
| `updated_files` | Must include every changed code file in the commit |

### Recommended fields (warn if missing, don't block)

| Field | Purpose |
|-------|---------|
| `patterns` | Pattern IDs involved (P1, P2, ...) |
| `not_updated_files` | Related files not changed, with justification |
| `tests_updated` | Test files created or modified |
| `docs_updated` | Documentation files created or modified |

### How enforcement works

1. **Pre-commit hook** (`pre-commit install` required): Runs `scripts/enforce_amendment_protocol.py` on every commit. Blocks if code files are staged without a valid amendment.

2. **CI check** (`.github/workflows/enforce-amendment.yml`): Runs on PRs to main. Uses `git diff` against `origin/main` to find all changed files and validates amendment coverage.

### Installing the pre-commit hook

```bash
# Install pre-commit (if not already installed)
uv tool install pre-commit

# Install the hook
cd <repo-root>
pre-commit install
```
